📋 New Spreadsheet Tabs Required

Add these tabs to your Google Sheets to enable the new features

Your Google Apps Script backend needs 4 new sheet tabs for the features added today. In your Google Sheet, create these tabs exactly as named below. Your Apps Script's doGet handler will automatically serve read/write/update/delete for any tab name — so no script changes are needed as long as the tab names match exactly.

Tab 1
Attendance
Stores attendance check, session scores and remarks per student per course.
id | group | studentId | colId | type | value | updatedAt
Tab 2
DistressReplies
Stores admin in-app replies to student "Talk to Coach" messages.
id | messageId | adminName | replyText | repliedAt
Tab 3
ReportChange
Stores student "Report a Change" posts visible to all admins.
id | studentId | studentName | changeType | message | postedAt
Tab 4
ReportChangeReplies
Stores admin replies to change reports, threaded per post.
id | postId | adminName | replyText | repliedAt
💡 Also: Add a cohort column to your Users tab

In the Users sheet, add a column header called cohort. Fill it with Junior Boys or Senior Boys for each student — or use the new Cohort button in the Users admin panel to cycle through values. This is what splits students into the two attendance lists.

⚡ Speed Tips — Make the App Blazing Fast
1. Apps Script — return JSON directly
In your doGet(), set ContentService.createTextOutput(JSON.stringify(data)).setMimeType(ContentService.MimeType.JSON) — never HTML output for data calls.

2. Apps Script — batch reads, not loops
Never call sheet.getRange(row,col) in a loop. Always use sheet.getDataRange().getValues() once and filter in JS.

3. Apps Script — use CacheService
Wrap heavy reads (Users, Courses) with CacheService.getScriptCache() with a 60-120s TTL. First call hits Sheets; subsequent calls return instantly.

const cache = CacheService.getScriptCache();
const cached = cache.get('Users');
if(cached) return JSON.parse(cached);
// ... read sheet ...
cache.put('Users', JSON.stringify(data), 120);

4. Apps Script — deploy as Web App with "Execute as: Me"
This avoids per-user auth overhead. Use "Anyone" access.

5. HTML side — Promise.all() for parallel reads
Already done in this app — never await sequential reads when they can run in parallel.

6. HTML side — in-memory 60s cache
Already built in (dbReadCached()). Use it for read-heavy views like attendance and grades.

7. Google Sheets — keep sheets lean
Archive old rows to a separate "Archive" sheet after each semester. The less data in active sheets, the faster every read.

8. Apps Script — avoid SpreadsheetApp.getActiveSpreadsheet()
Use SpreadsheetApp.openById('YOUR_SHEET_ID') instead — it's significantly faster in web app deployments.
